Task 3: Testing DNS Resolution

1. Restart BIND9

Command:

sudo systemctl restart bind9

Explanation: After configuring forward and reverse zones, restart the BIND9 service to apply the changes. Ensure there are no syntax errors in the configuration by running sudo named-checkconf first.

2. Test Forward DNS Resolution

Command:

dig example.com @localhost

Explanation: This command uses dig to test whether BIND9 can resolve the domain "example.com" to an IP address. The @localhost part tells the command to query the local BIND9 server.

3. Test Reverse DNS Resolution

Command:

dig -x 192.168.1.10 @localhost

Explanation: This command tests reverse DNS resolution, which maps an IP address (in this case, 192.168.1.10) back to a domain name. This checks whether the reverse zone is correctly configured.

4. Verify DNS Query Logs

Command:

sudo tail -f /var/log/syslog

Explanation: Viewing the logs can help troubleshoot issues. When DNS queries are made, BIND9 logs information in /var/log/syslog. Use tail -f to follow the logs in real time.
