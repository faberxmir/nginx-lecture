Task 1: Installing and Setting Up BIND9

1. Install BIND9

Command:

sudo apt update
sudo apt install bind9 bind9utils bind9-doc

Explanation: This command installs BIND9 and its utilities on your Ubuntu server. BIND9 is a widely-used DNS software, and the utilities will help you manage and troubleshoot DNS issues.

2. Start and Enable BIND9

Command:

sudo systemctl start bind9
sudo systemctl enable bind9

Explanation: This will start the BIND9 service and ensure that it starts automatically when the system reboots. Managing BIND9 through systemctl is similar to controlling other services like Nginx.

3. Check BIND9 Status

Command:

sudo systemctl status bind9

Explanation: Checking the status of the BIND9 service ensures it is running correctly and shows logs that could be helpful if there are any issues.

4. Configure BIND9 to Listen on the Network

Command:

sudo nano /etc/bind/named.conf.options

Add the following configuration under options:

options {
    directory "/var/cache/bind";

    recursion yes;
    allow-query { any; };
    forwarders {
        8.8.8.8;  // Google Public DNS
        8.8.4.4;  // Google Public DNS
    };
};

Explanation: This configuration allows the BIND9 server to forward DNS queries it cannot resolve to Google's public DNS. It also enables recursion, allowing your DNS server to resolve queries for clients.
